## 🔗 RELATIONSHIP DESIGN

The system models real-world relationships:

- 👨‍🎓 Student ↔ 📚 Course → ENROLL  
- 👨‍🎓 Student ↔ 🤝 Club → JOIN  
- 👨‍🎓 Student ↔ 📖 Book → BORROW  
- 👨‍🏫 Professor ↔ 👨‍🎓 Student → COLLABORATE  
- 🏛️ Department ↔ 🔬 Research Project  

All relationships are implemented using **Primary & Foreign Keys**.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 🧱 NORMALIZATION (3NF)

✔ 1NF → Atomic values ensured  
✔ 2NF → No partial dependency  
✔ 3NF → No transitive dependency  

Result: clean, consistent, and scalable database design.
